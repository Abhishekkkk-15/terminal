# Build a Warp-Inspired Terminal UI



Build a modern desktop terminal frontend inspired by Warp, Cursor, and Claude Code.



IMPORTANT:

This is NOT a chatbot application.

This is NOT a command prompt clone.

This is a terminal emulator frontend that will connect to a Rust backend through IPC/WebSocket events.



Tech Stack:



* React

* TypeScript

* TailwindCSS

* shadcn/ui

* xterm.js

* Zustand for state management



Design Requirements:



* Dark theme by default

* Professional developer-tool aesthetic

* Similar visual quality to Warp, Cursor, Raycast, and Linear

* Smooth animations

* Rounded corners

* Modern typography

* Minimalistic UI

* Responsive desktop layout

Desktop Framework Requirements



This application MUST be built as a Tauri desktop application.



Tech Stack:



* Tauri v2

* React

* TypeScript

* TailwindCSS

* shadcn/ui

* Zustand

* xterm.js



Architecture Requirements:



The Rust backend will be implemented separately.



The frontend must be designed specifically for Tauri integration.



Do NOT use:



* Express

* Node backend servers

* Next.js API routes

* WebSocket localhost servers

* Electron



Assume all backend communication happens through Tauri IPC.



Create abstraction layers for:



* invoke()

* listen()

* emit()



Backend communication should be isolated inside:



src/services/tauri/



Example structure:



src/

├── services/

│   ├── tauri/

│   │   ├── terminal.ts

│   │   ├── workspace.ts

│   │   ├── ai.ts

│   │   └── events.ts



The UI must never directly call Tauri APIs.



Use service abstractions so the Rust backend can be connected later without changing UI components.



Assume the Rust backend will expose commands such as:



invoke("write_terminal")

invoke("resize_terminal")

invoke("get_workspace")

invoke("execute_workflow")

invoke("send_agent_message")



Assume Rust backend events:



terminal-output

command-started

command-finished

workspace-updated

agent-message

backend-status



Design the frontend around these events.



The application should feel like a native desktop application rather than a website.



Use resizable panels.



Use desktop-style layouts.



Support keyboard-first navigation.



Avoid browser-like navigation patterns.



The terminal should be the primary focus of the interface.
